**backend/alembic/versions/001_initial_schema.py:**
Revision ID: `001_initial_schema`
Down revision: `None`

`upgrade()` function MUST execute in this exact order:

1. Enable extensions:
   ```sql
   CREATE EXTENSION IF NOT EXISTS pg_trgm;
   CREATE EXTENSION IF NOT EXISTS pgcrypto;
   ```

2. Create `feed_runs` table FIRST (because `ioc_sources` references it):
   - All columns as specified in the ORM model
   - Indexes: `feed_runs_feed_name_idx` on (feed_name), `feed_runs_started_at_idx` on (started_at DESC)

3. Create `iocs` table:
   - All columns as specified in the ORM model
   - UNIQUE constraint `uq_iocs_value_type` on (value, type)
   - Generated column: `ts_vector TSVECTOR GENERATED ALWAYS AS (to_tsvector('english', value || ' ' || COALESCE(metadata::text, ''))) STORED`
   - Indexes:
     - `iocs_ts_vector_idx` GIN on ts_vector
     - `iocs_value_trgm_idx` GIN on value using gin_trgm_ops
     - `iocs_type_idx` B-tree on (type)
     - `iocs_severity_idx` B-tree on (severity DESC)
     - `iocs_last_seen_idx` B-tree on (last_seen DESC)
     - `iocs_type_severity` B-tree composite on (type, severity DESC)
     - `iocs_type_last_seen` B-tree composite on (type, last_seen DESC)
     - `iocs_active_idx` partial B-tree on (is_active) WHERE is_active = TRUE

4. Create `ioc_sources` table:
   - FK to iocs(id) ON DELETE CASCADE
   - FK to feed_runs(id)
   - Indexes: `ioc_sources_ioc_id_idx`, `ioc_sources_feed_name_idx`, `ioc_sources_ingested_at_idx` DESC

5. Create `ioc_relationships` table:
   - Both FKs to iocs(id) ON DELETE CASCADE
   - UNIQUE constraint `uq_ioc_relationship` on (source_ioc, target_ioc, relationship)
   - Indexes: `ioc_rel_source_idx` on source_ioc, `ioc_rel_target_idx` on target_ioc

6. Create workspace tables (`tags`, `notes`, `watchlists`):
   - All with `user_id UUID NOT NULL`
   - tags: UNIQUE `uq_tag_per_user_ioc` on (ioc_id, user_id, tag), indexes on ioc_id and user_id
   - notes: indexes on ioc_id and user_id
   - watchlists: UNIQUE `uq_watchlist_user_ioc` on (user_id, ioc_id), indexes on user_id and ioc_id

`downgrade()` function: Drop all tables in reverse order, then drop extensions.

NOTE: Use `op.execute()` for raw SQL (extensions, generated columns, partial indexes) and `op.create_table()` / `op.create_index()` for standard DDL. The generated column and partial index require raw SQL because Alembic's op helpers do not support GENERATED ALWAYS AS or WHERE clauses natively.
